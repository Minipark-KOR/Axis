# Token Estimation Specification

## 0. Scope

본 문서는 Axis 시스템에서 사용하는 **토큰 수치의 관측 규약**을 정의한다.
여기서 토큰 수치는 모델 호출의 실제 발생 여부와 무관하게,
**비교 가능성과 장기 관측**을 위해 계측되는 추정 값이다.

이 문서는 다음을 정의한다.
- 토큰 수치가 언제, 어떤 단계에서 관측되는가
- 토큰 수치가 어떤 성격의 데이터로 취급되는가
- 토큰 추정 방식의 식별과 안정성 규칙

이 문서는 다음을 정의하지 않는다.
- 토큰 계산 알고리즘 또는 코드
- 특정 모델 또는 벤더의 구현 세부
- 비용 절감 전략 또는 최적화 판단
- 실시간 제어 규칙

토큰 수치는 판단 기준이 아니라 **관측 지표**다.

## 1. Nature of Token Metrics

Axis에서 토큰 수치는 정확한 사용량을 보장하지 않는다.
토큰 수치는 **추정치(estimate)**이며,
비교와 경향 분석을 위한 관측값으로만 사용된다.

토큰 수치는 다음 목적에 한해 사용된다.
- 처리 단계 간 데이터 양의 비교
- 기간별 변화 추이 분석
- 장기 전략 검토를 위한 근거 제공

토큰 수치는 다음 용도로 사용되지 않는다.
- 실행의 즉시 중단 또는 허용 판단
- 자동 확장 또는 자동 축소
- 상태 판정 또는 등급화

## 2. Observation Points

토큰 수치는 다음 지점에서 관측될 수 있다.
각 관측 지점은 서로 다른 맥락을 가지며,
값의 직접 비교는 동일 지점 내에서만 허용된다.

### 2.1 Raw Input Stage

- 정리 이전 원본 데이터 기준 추정 토큰 수
- 입력 데이터의 잠재적 규모를 관측
- 실제 전송과 무관

### 2.2 Processed Data Stage

- 내부 정리 및 분리 이후 기준 추정 토큰 수
- 구조 변경에 따른 양의 변화를 관측
- 내부 메타 포함 가능

### 2.3 Transmission Candidate Stage

- 외부 전송 대상이 될 수 있는 최종 데이터 기준 추정 토큰 수
- 내부 사고(reasoning) 및 메타가 제거된 상태
- 실제 전송 여부와 독립적

## 3. Estimation Method Identifier

각 토큰 수치는 반드시 **추정 방식 식별자(method identifier)**와 함께 기록된다.

- method identifier는 계산 규약의 버전을 식별한다
- 동일한 identifier는 동일한 추정 전제를 의미한다
- identifier가 다른 수치는 직접 비교 대상이 아니다

계산 규약의 상세(수식, 계수, 모델 종속 요소)는
본 문서의 범위를 벗어나며,
별도의 기술 문서에서 관리된다.

## 4. Accuracy and Stability

토큰 수치는 다음 원칙을 따른다.

- 정확도를 보장하지 않는다
- 일관성을 우선한다
- 시간이 지나도 비교 가능해야 한다

새로운 계산 규약이 도입될 수 있으나,
기존 데이터의 토큰 수치는 재작성되지 않는다.
과거 데이터는 당시의 관측 사실로 유지된다.

## 5. Relationship to Budget and Cost

토큰 수치는 예산 및 비용 논의의 입력 자료로 사용될 수 있다.
그러나 다음 원칙은 반드시 유지된다.

- 토큰 수치는 직접적인 비용 값을 의미하지 않는다
- 토큰 수치만으로 실행을 제어하지 않는다
- 예산 판단은 토큰 수치를 포함한 복수 관측에 기반해
  **사람**에 의해 이루어진다

토큰 수치는 결정이 아니라 **설명 가능한 근거**다.

## 6. Forbidden Interpretations

다음 해석은 금지된다.

- 토큰 수치 감소를 효율 개선으로 단정하는 해석
- 토큰 수치를 품질 또는 가치 지표로 사용하는 행위
- 토큰 수치를 기준으로 한 자동 조정 로직

토큰 수치는 결과가 아니라 **현상 기록**이다.
