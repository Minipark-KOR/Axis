# Data Schema

## 0. Scope

본 문서는 Axis 시스템에서 다루는 데이터의 **구조적 골격**을 정의한다.
여기서 말하는 데이터는 다음을 모두 포함한다.

- 원본 데이터 (RAW)
- 처리 및 분석 과정에서 생성된 데이터
- 시스템이 관측을 위해 부여한 메타 정보(meta)

본 문서는 다음을 **정의하지 않는다**.

- 데이터의 의미 해석
- 데이터의 중요도 또는 가치 판단
- 운영 판단 기준
- 저장 매체 또는 인프라 구성

이 문서는 분석과 판단을 가능하게 하기 위한 **형태(shape)**만을 고정한다.

## 1. Data vs Meta

Axis 시스템은 데이터와 메타 정보를 명확히 구분한다.

### 1.1 Data

Data는 시스템 밖에서도 의미를 가지는 정보이며,
해석의 대상이 되는 내용 자체를 의미한다.

예:
- 텍스트 본문
- 모델 입력 메시지
- 계산 결과 값
- 정제된 레코드

Data는 수정, 재해석, 재결합의 대상이 될 수 있다.

### 1.2 Meta

Meta는 데이터의 의미를 설명하지 않는다.
Meta는 데이터의 **관측 가능한 속성**만을 기록한다.

Meta는 아래 조건을 모두 만족해야 한다.

- 데이터의 내용을 변경하지 않는다
- 다른 데이터와 비교 가능하다
- 사람이 다시 계산하거나 재구성할 수 있다

Meta는 판단이 아닌 **좌표계**다.

## 2. Meta Categories

Axis에서 허용되는 Meta는 다음 범주로 한정된다.

### 2.1 Volume Metrics

- 문자 수
- 바이트 수
- 토큰 수 (추정치 포함)
- 항목 개수
- 메시지 개수

이 값들은 데이터의 **양**을 표현하며,
의미 또는 품질을 나타내지 않는다.

### 2.2 Time Metrics

- 생성 시각
- 처리 시작 시각
- 처리 종료 시각
- 단계 간 경과 시간

시간 메타는 처리 리듬과 흐름을 관측하기 위한 용도로만 사용된다.

### 2.3 Process State

- 처리됨 / 스킵됨
- 스킵 사유 코드
- 처리 단계 식별자 (예: OCI, Neuron, Engram)

상태 메타는 평가를 포함하지 않는다.
성공·실패·정상·비정상이라는 표현을 사용하지 않는다.

## 3. Forbidden Meta

다음 정보는 어떤 형태로도 Meta로 포함될 수 없다.

- 데이터의 요약, 결론, 핵심 문장
- 효율, 절약, 최적과 같은 평가 표현
- 인과 관계 서술
- 권고 또는 행동 유도 정보
- 자동 판단에 사용될 수 있는 점수 또는 등급

이러한 정보는 반드시 **외부 문서 또는 사람의 판단 영역**에 존재해야 한다.

## 4. Meta Stability Rule

Meta의 구조는 장기 비교를 위해 가능한 한 안정적으로 유지되어야 한다.
새로운 Meta 필드의 추가는 허용되나,
기존 Meta의 의미 변경은 허용되지 않는다.

과거 데이터는 당시의 관측 사실로 존중되며,
후속 구현에 의해 재해석되지 않는다.
