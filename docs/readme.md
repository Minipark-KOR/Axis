# Project The Axis

**The Axis**는 Neural Data Architecture(NDA)를 기반으로 한  
**사람의 판단을 보호하기 위한 분석·운영 시스템**이다.

Axis는 자동 판단 시스템이 아니다.  
결론을 생성하지 않고, 행동을 추천하지 않으며,  
사람이 다시 판단할 수 있도록 **사실의 기록만 남긴다**.

---

## 1. Axis가 풀고자 하는 문제

Axis는 다음 질문에서 출발한다.

- 제한된 예산 안에서 AI 분석을 지속하려면 어떻게 운영해야 하는가?
- 실행 속도와 데이터 품질의 균형은 누가, 언제 판단해야 하는가?
- 자동화가 판단을 대신하지 않도록 하려면 구조를 어떻게 고정해야 하는가?

Axis의 답은 하나다.

> **판단을 자동화하지 않고,  
> 판단의 근거가 왜곡되지 않도록 구조를 설계한다.**

---

## 2. 핵심 전제 (Immutable Assumptions)

Axis는 다음 전제를 절대 변경하지 않는다.

- 시스템은 결론을 내리지 않는다
- 시스템은 인과를 추론하지 않는다
- 시스템은 행동을 권장하지 않는다
- 모든 운영 판단의 책임은 사용자에게 있다

따라서 Axis가 제공하는 모든 정보는  
**정확하게 판단하기 위한 근거(view)**다.

---

## 3. Axis의 역할 분리

Axis는 다음 다섯 역할을 명확히 분리한다.

- **Synapse**  
  외부 데이터 수집과 1차 정제만 수행한다

- **Neuron**  
  짧은 시간의 계산만 수행하고 즉시 종료된다

- **Cortex**  
  결과와 그 사이의 관계를 *해석 없이* 축적한다

- **Engram**  
  정제된 기록을 장기 보관한다

- **User**  
  무엇을 연결하고, 무엇을 해석할지 판단하는 유일한 주체다

이 경계는 자동화 요구나 편의성을 이유로 침범되지 않는다.

---

## 4. 제어와 실행의 분리

Axis는 제어(Control)와 실행(Execution)을 물리적으로 분리한다.

- 제어 계층은 **항상 켜져 있으며**
- 실행 계층은 **필요할 때만 생성되고 즉시 종료된다**

이 구조는 성능 최적화가 목적이 아니라,
**운영 통제와 비용 예측 가능성을 보장하기 위한 선택**이다.

---

## 5. 실패에 대한 관점

Axis에서 실패는 예외가 아니다.

- 단일 실행 실패는 정상 상태다
- 실패는 제거 대상이 아니라 관측 신호다
- 반복 패턴만이 해석의 대상이 된다

**실패를 기록으로 남긴다.**

---

## 6. 자동화를 제한하는 이유

Axis는 더 많은 자동화를 목표로 하지 않는다.

- 자동 판단
- 자동 조정
- 자동 확장
- 점수 기반 상태 판정

이 모든 것은 **판단 책임을 시스템으로 이동시키는 통로**이기 때문에  
의도적으로 설계에서 배제된다.

> Axis는 판단을 제거하기 위해 자동화를 쓰지 않는다.  
> Axis는 판단을 보호하기 위해 자동화를 제한한다.
